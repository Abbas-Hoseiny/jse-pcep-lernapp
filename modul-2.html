Variablen, Datentypen & Type Casting

Das wichtigste Kapitel. var vs let vs const, primitive Typen, typeof, Umwandlung.

2.1 Variablen deklarieren

Eine Variable ist ein benannter Speicherplatz für einen Wert. In modernem JS gibt es drei Schlüsselwörter:

var   alterAlt = 30;     // alte Syntax (bis ES5)
let   alter = 25;        // moderne, blockskopierte Variable
const PI = 3.14;         // Konstante (nicht neu zuweisbar)

Unterschiede im Überblick

Eigenschaft var let const
ScopeFunktionBlockBlock
Neu zuweisbarJaJaNein
Neu deklarierbarJaNeinNein
Hoisting (Wert)undefinedTDZ-FehlerTDZ-Fehler
Faustregel: Nutze const standardmäßig, let wenn du neu zuweisen musst, var nie mehr.

2.2 Primitive Datentypen

JavaScript hat 7 primitive Typen:

  • Number — Zahlen (Ganzzahl & Gleitkomma): 42, 3.14, -7
  • BigInt — sehr große Ganzzahlen: 9007199254740993n
  • String — Text: "hi", 'hi', `hi`
  • Boolean — Wahrheitswerte: true, false
  • undefined — Variable wurde deklariert aber nicht initialisiert
  • null — bewusst „kein Wert"
  • Symbol — einzigartiger, unveränderlicher Wert (selten)

Alles andere (Array, Object, Function, Date…) ist ein Object.

2.3 Der Number-Typ

let ganz = 42;
let gleit = 3.14;
let negativ = -17;
let expo = 1.5e3;        // 1500
let hex = 0xff;          // 255
let bin = 0b1010;        // 10
let oct = 0o17;          // 15

// Spezielle Werte:
Infinity                 // 1 / 0
-Infinity                // -1 / 0
NaN                      // "Not a Number" — z. B. 0/0, "abc" * 2

Alle Zahlen sind 64-bit-Gleitkomma (IEEE 754). Das führt zu:

0.1 + 0.2 === 0.3   // false !
0.1 + 0.2           // 0.30000000000000004
Achtung: NaN === NaN ist false. Prüfe mit Number.isNaN(x) oder Number.isNaN().

Nützliche Number-Methoden

(3.14159).toFixed(2);       // "3.14"
Number.isInteger(5);        // true
Number.isNaN(NaN);          // true
Number.parseFloat("3.14px"); // 3.14
Number.parseInt("42abc", 10); // 42
Math.max(1, 2, 3);          // 3
Math.floor(3.9);            // 3
Math.round(3.5);            // 4

2.4 Der String-Typ

let a = "doppelt";
let b = 'einfach';
let c = `template`;   // Backticks — erlauben Interpolation

// Template Literals:
let name = "Ali";
let gruss = `Hallo, ${name}!`;       // "Hallo, Ali!"
let multi = `Zeile 1
Zeile 2`;                             // mehrzeilig erlaubt

// Escape-Sequenzen:
"Zeile1\nZeile2"    // \n = Zeilenumbruch
"Tab\there"          // \t = Tab
"Er sagte \"Hi\""   // \" = Anführungszeichen
"Backslash: \\"     // \\ = Backslash

Nützliche String-Methoden & Properties

"hallo".length;              // 5
"hallo".toUpperCase();        // "HALLO"
"HALLO".toLowerCase();        // "hallo"
"hallo".charAt(1);            // "a"
"hallo"[1];                   // "a"
"hallo welt".indexOf("welt"); // 6
"hallo".includes("all");      // true
"hallo".slice(1, 4);          // "all"
"  hi  ".trim();              // "hi"
"a,b,c".split(",");           // ["a", "b", "c"]
"ha".repeat(3);               // "hahaha"
"hallo".replace("h", "b");    // "ballo"
Strings sind unveränderlich (immutable). s.toUpperCase() verändert s nicht, sondern gibt einen neuen String zurück.

2.5 Der Boolean-Typ

Nur zwei Werte: true oder false. Entsteht meist durch Vergleiche.

let istOk = true;
let istLeer = (name === "");  // true oder false

// "Truthy" / "Falsy":
// Falsy: false, 0, "", null, undefined, NaN, 0n
// Alles andere ist truthy!
Boolean(0);        // false
Boolean("");       // false
Boolean("hallo");  // true
Boolean([]);       // true  (leeres Array ist truthy!)
Boolean({});       // true  (leeres Objekt ist truthy!)

2.6 undefined und null

let x;                // undefined (deklariert, kein Wert)
let y = null;         // null (bewusst leer)

typeof undefined;     // "undefined"
typeof null;          // "object"  ← historisches Kuriosum!

undefined == null;    // true  (lose Gleichheit)
undefined === null;   // false (strikte Gleichheit)
Merkspruch: undefined = „JavaScript hat ihn nicht gesetzt". null = „Entwickler hat ihn absichtlich auf leer gesetzt".

2.7 typeof — den Typ prüfen

typeof 42;            // "number"
typeof "hi";          // "string"
typeof true;          // "boolean"
typeof undefined;     // "undefined"
typeof null;          // "object"       ← Stolperfalle!
typeof {};            // "object"
typeof [];            // "object"       (Arrays sind Objekte)
typeof function(){};  // "function"
typeof 42n;           // "bigint"
typeof Symbol();      // "symbol"

2.8 Typumwandlung (Type Casting / Coercion)

Explizit — du rufst den Konverter auf

String(42);         // "42"
String(true);       // "true"
Number("42");       // 42
Number("abc");      // NaN
Number(true);       // 1
Number(false);      // 0
Number("");         // 0
Number(null);       // 0
Number(undefined);  // NaN
Boolean(1);         // true
Boolean(0);         // false
parseInt("12px");   // 12
parseFloat("3.14"); // 3.14

Implizit — JS wandelt automatisch

"5" + 1;       // "51"   ← String gewinnt bei +
"5" - 1;       // 4      ← - erzwingt Number
"5" * "2";     // 10
true + 1;      // 2      ← true → 1
null + 1;      // 1      ← null → 0
undefined + 1; // NaN    ← undefined → NaN
"" + 0;        // "0"
[] + [];       // ""     ← Kurios!
Faustregel: Bei + gewinnt der String. Bei -, *, / wird zu Number umgewandelt. Nutze === statt ==, um Coercion zu vermeiden.
