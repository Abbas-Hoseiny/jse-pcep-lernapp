Datenstrukturen: Listen, Tupel, Dicts, Strings

Erstellen, zugreifen, slicen, Methoden, iterieren, 2D.

5.1 Listen (list) — veränderlich

leer = []
zahlen = [1, 2, 3]
gemischt = [1, "hi", True, None, [4, 5]]

# Zugriff (Index ab 0):
zahlen[0]      # 1
zahlen[-1]     # 3   — negativer Index von hinten
zahlen[99]     # IndexError!

# Länge:
len(zahlen)    # 3

# Ändern:
zahlen[0] = 99
# zahlen → [99, 2, 3]

Listen sind mutable (veränderlich) und können beliebige Typen mischen.

5.2 Slicing — [start:stop:step]

l = [0, 1, 2, 3, 4, 5]
l[1:4]       # [1, 2, 3]         start inklusiv, stop exklusiv
l[:3]        # [0, 1, 2]         von Anfang
l[3:]        # [3, 4, 5]         bis Ende
l[::2]       # [0, 2, 4]         jedes 2.
l[::-1]      # [5, 4, 3, 2, 1, 0]  umgekehrt
l[-3:]       # [3, 4, 5]         letzte 3

# Slicing gibt KOPIE zurück:
kopie = l[:]
Wichtig: Slicing funktioniert genauso für Strings und Tupel.

5.3 Listen-Methoden

l = [2, 3]
l.append(4)        # l = [2, 3, 4]          am Ende anfügen
l.insert(0, 1)     # l = [1, 2, 3, 4]       an Position einfügen
l.extend([5, 6])   # l = [1, 2, 3, 4, 5, 6] mehrere anfügen
l.remove(3)        # entfernt erste 3
l.pop()             # entfernt & gibt letztes zurück
l.pop(0)            # entfernt & gibt erstes zurück
l.index(2)          # gibt Index von 2 zurück (ValueError wenn fehlt)
l.count(2)          # zählt Vorkommen
l.sort()            # sortiert IN PLACE
l.sort(reverse=True)
l.reverse()         # kehrt IN PLACE um
l.clear()           # leert die Liste
l.copy()            # flache Kopie

# Built-in-Funktionen:
len(l)
sum(l)               # nur für Zahlen
max(l), min(l)
sorted(l)            # gibt neue sortierte Liste (Original bleibt)
reversed(l)          # Iterator
list(range(5))       # [0,1,2,3,4]
In-place vs. neue Liste:
  • l.sort() ändert die Liste, gibt None zurück!
  • sorted(l) gibt eine neue Liste zurück.
  • Gleiches für .reverse() vs reversed().

5.4 del-Statement

l = [10, 20, 30, 40]
del l[1]          # [10, 30, 40]
del l[1:3]        # [10]
del l             # Variable komplett löschen

5.5 Mehrdimensionale Listen

matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]
matrix[0][0]    # 1
matrix[2][1]    # 8

for zeile in matrix:
    for wert in zeile:
        print(wert, end=" ")
    print()
Falle: [[0]*3]*3 erzeugt 3× die gleiche Referenz! 
m = [[0]*3]*3
m[0][0] = 1
print(m)   # [[1,0,0],[1,0,0],[1,0,0]] — alle Zeilen geändert!
Korrekt: [[0]*3 for _ in range(3)]

5.6 Tupel (tuple) — unveränderlich

leer = ()
einer = (1,)        # Komma erforderlich!
ohne_komma = (1)    # ist int, KEIN Tupel!
mehrere = (1, 2, 3)
auch_tupel = 1, 2, 3  # Klammern optional
punkt = 3, 4        # Tuple-Packing

x, y = punkt        # Tuple-Unpacking
a, b, c = 1, 2, 3

# Zugriff wie Liste, aber kein Ändern:
mehrere[0]          # 1
# mehrere[0] = 99   # TypeError!

Tupel-Methoden (nur 2!)

t = (1, 2, 2, 3)
t.count(2)    # 2
t.index(3)    # 3

Tupel sind schneller als Listen und nutzbar als Dict-Keys. Listen nicht!

5.7 Dictionaries (dict) — Schlüssel-Wert-Paare

d = {"name": "Ali", "alter": 25}

# Zugriff:
d["name"]           # 'Ali'
d["fehlt"]           # KeyError!
d.get("fehlt")      # None (kein Error)
d.get("fehlt", 0)   # Default 0

# Ändern / hinzufügen:
d["name"] = "Bob"
d["stadt"] = "Berlin"

# Entfernen:
del d["alter"]
d.pop("stadt")
d.clear()

# Prüfen:
"name" in d         # True  — prüft KEYS
"Ali" in d.values()

# Durchlaufen:
for key in d:                       # Keys
    print(key)
for key, value in d.items():        # Paare
    print(key, value)
for value in d.values():
    print(value)
Regeln für Keys: müssen hashable sein → int, str, tuple (von hashables) OK. List, dict, set NICHT.

5.8 Dict-Methoden

d.keys()       # dict_keys([...])
d.values()     # dict_values([...])
d.items()      # dict_items([(k,v), ...])
d.get(k)       # Value oder None
d.get(k, def)  # Value oder def
d.pop(k)       # entfernt & gibt zurück
d.update({"x": 1})  # fügt hinzu / überschreibt
d.copy()       # flache Kopie
len(d)         # Anzahl Paare

5.9 Strings als Sequenzen

Strings in Python verhalten sich wie unveränderliche Sequenzen.

s = "Python"
s[0]          # 'P'
s[-1]         # 'n'
s[1:4]        # 'yth'
s[::-1]       # 'nohtyP'
len(s)        # 6
"P" in s       # True

# ord / chr (wichtig für PCEP!)
ord("A")      # 65
chr(65)       # 'A'

Wichtige String-Methoden

"hallo".upper()         # 'HALLO'
"HALLO".lower()          # 'hallo'
"hi welt".title()        # 'Hi Welt'
"hi".capitalize()        # 'Hi'
"hi".swapcase()          # 'HI'
"  hi  ".strip()         # 'hi'
"abc".replace("a", "X")  # 'Xbc'
"a,b,c".split(",")       # ['a','b','c']
",".join(["a","b","c"])  # 'a,b,c'
"abc".startswith("ab")   # True
"abc".endswith("bc")     # True
"abc".find("b")          # 1       — -1 wenn fehlt
"abc".index("b")          # 1       — ValueError wenn fehlt
"a".isalpha()            # True
"1".isdigit()            # True
"a1".isalnum()           # True
"ABC".isupper()          # True
" ".isspace()            # True
"abc".center(7, "*")     # '**abc**'
"abc".ljust(7, "-")      # 'abc----'
"abc".rjust(7, "-")      # '----abc'
"5".zfill(3)             # '005'
Alle String-Methoden geben einen neuen String zurück — Strings sind immutable!

5.10 Kopieren — flach vs. tief

a = [1, [2, 3]]
b = a             # SELBE Referenz!
c = a[:]          # flache Kopie
d = a.copy()      # flache Kopie
import copy
e = copy.deepcopy(a)  # tiefe Kopie

a[0] = 99
print(a, b, c)    # a und b geändert, c original
a[1].append(4)
print(c)          # [1, [2, 3, 4]] — innere Liste geteilt!
print(e)          # [1, [2, 3]]    — tiefe Kopie unabhängig
