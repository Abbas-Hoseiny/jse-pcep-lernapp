Operatoren

Arithmetisch, Vergleich, Logik, Bitweise, Identität, Membership, Präzedenz.

3.1 Arithmetische Operatoren

5 + 2       # 7    Addition
5 - 2       # 3    Subtraktion
5 * 2       # 10   Multiplikation
5 / 2       # 2.5  Division (IMMER float!)
5 // 2      # 2    Ganzzahl-Division (floor)
5 % 2       # 1    Modulo (Rest)
5 ** 2      # 25   Potenz
-5          # Negation
+5          # Plus (selten genutzt)
Wichtig für PCEP:
  • / gibt immer float zurück: 4 / 2 == 2.0 (nicht 2!)
  • // ist Floor-Division (rundet ab, auch bei negativen Zahlen): -7 // 2 == -4 (nicht -3!)
  • ** ist Potenz, nicht XOR
10 / 3      # 3.3333333333333335
10 // 3     # 3
-10 // 3    # -4   ← floor, nicht truncation!
10.0 // 3   # 3.0  — wenn ein float dabei ist, bleibt float
10 % 3      # 1
-10 % 3     # 2    ← Vorsicht: Python-Modulo folgt dem Vorzeichen des Divisors

3.2 Zuweisungsoperatoren

x = 10

x += 5     # x = x + 5   → 15
x -= 3     # x = x - 3   → 12
x *= 2     # x = x * 2   → 24
x /= 4     # x = x / 4   → 6.0  (wird float!)
x //= 2    # x = x // 2  → 3.0
x %= 2     # x = x % 2   → 1.0
x **= 3    # x = x ** 3  → 1.0
x &= 5     # bitweise
x |= 5
x ^= 5
x <<= 1
x >>= 1

3.3 Vergleichsoperatoren

5 == 5         # True
5 != 3         # True
5 > 3          # True
5 < 3          # False
5 >= 5         # True
5 <= 4         # False

# String-Vergleich (lexikographisch, Unicode):
"apfel" < "banane"    # True
"B" < "a"              # True ('B'=66, 'a'=97)

# Chained Comparisons — Python-Spezialität!
1 < x < 10             # äquivalent zu: 1 < x and x < 10
0 <= alter < 18        # Minderjährig?
Nur ==, kein ===! Python vergleicht immer strikt nach Wert — Typ wird nicht umgewandelt.

3.4 Logische Operatoren

In Python heißen sie and, or, not — ausgeschrieben, nicht &&/||/!.

True and False     # False
True or False      # True
not True           # False

# Short-Circuit wie in JS:
# and gibt den ersten falsy Wert oder den letzten
True and "hallo"   # 'hallo'
0 and "hallo"      # 0
"a" and "b" and "c"  # 'c'

# or gibt den ersten truthy Wert oder den letzten
None or "default"  # 'default'
0 or "" or "x"     # 'x'
"a" or "b"         # 'a'

3.5 Bitweise Operatoren

5 & 3     # 1    AND
5 | 3     # 7    OR
5 ^ 3     # 6    XOR
~5        # -6   NOT (bitweise Inversion)
5 << 1    # 10   Left-Shift (× 2)
5 >> 1    # 2    Right-Shift (÷ 2)

Selten genutzt, aber PCEP fragt danach. Merke: ^ in Python ist XOR, nicht Potenz (das ist **).

3.6 Identitäts-Operatoren: is / is not

x = [1, 2]
y = x          # gleiche Referenz
z = [1, 2]    # gleicher Wert, andere Referenz

x == y         # True  (gleicher Wert)
x == z         # True
x is y         # True  (gleiches Objekt)
x is z         # False (verschiedene Objekte)

# Standard-Fall: is bei None
if x is None:
    pass
Faustregel: is nur für None, True, False. Für Wertvergleiche immer ==.

3.7 Membership-Operatoren: in / not in

3 in [1, 2, 3]           # True
5 in [1, 2, 3]           # False
"a" in "banana"           # True
"x" not in "banana"       # True
"name" in {"name": "Ali"} # True  (prüft KEYS bei dicts)

3.8 Der „walrus" Operator := (ab 3.8)

if (n := len([1,2,3])) > 2:
    print(f"Liste hat {n} Elemente")

Weist zu und gibt gleichzeitig den Wert zurück. Für PCEP nicht kritisch, aber gut zu kennen.

3.9 String-Operatoren

"Hallo" + " " + "Welt"    # 'Hallo Welt'   Konkatenation
"ab" * 3                   # 'ababab'       Wiederholung
"a" in "banana"            # True
"e" not in "banana"        # True

Achtung: "5" + 3 wirft TypeError — explizit konvertieren: "5" + str(3).

3.10 Operator-Präzedenz (von hoch nach niedrig)

  1. () Klammern, Literale
  2. ** Potenz (rechtsassoziativ!)
  3. unär +x, -x, ~x
  4. *, /, //, %
  5. +, - (binär)
  6. Shifts <<, >>
  7. Bitweise &
  8. Bitweise ^
  9. Bitweise |
  10. Vergleiche ==, !=, <, >, in, is
  11. not
  12. and
  13. or
  14. Ternär a if b else c
Spezial: ** ist rechtsassoziativ!
2 ** 3 ** 2 == 2 ** (3 ** 2) == 2 ** 9 == 512

3.11 Ternärer Ausdruck

Python hat kein ? :, aber eine andere Syntax:

status = "erwachsen" if alter >= 18 else "minderjährig"
