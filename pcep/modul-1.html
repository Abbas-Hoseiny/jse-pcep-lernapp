Python-Grundlagen

Interpreter, Syntax, Indentation, print(), Kommentare, Literale, Numeralsysteme, Keywords.

1.1 Was ist Python?

Python ist eine high-level, interpretierte, dynamisch typisierte, stark typisierte Programmiersprache. Entwickelt 1989/1991 von Guido van Rossum.

  • Interpretiert: Code wird zur Laufzeit ausgeführt (kein manueller Kompilierschritt).
  • Dynamisch typisiert: Variablen haben keinen festen Typ.
  • Stark typisiert: Typen werden nicht automatisch gemischt ("5" + 3 wirft Fehler!).
  • Multi-Paradigma: prozedural, objektorientiert, funktional.
  • Open Source, plattformunabhängig.

1.2 Compilation vs. Interpretation

  • Compiler: Übersetzt den gesamten Quellcode einmal in Maschinencode (.exe). Schnelle Ausführung, aber nicht plattformunabhängig. Beispiele: C, C++, Go.
  • Interpreter: Liest und führt den Code Zeile für Zeile aus. Langsamer, aber portabel. Python, JavaScript, Ruby.

Python nutzt beides: Der Quellcode wird zuerst in Bytecode (.pyc) kompiliert und dann von der Python Virtual Machine (PVM) interpretiert.

1.3 Lexis, Syntax, Semantik

  • Lexis: Das Alphabet der Sprache — Tokens (Keywords, Operatoren, Bezeichner).
  • Syntax: Die Grammatik — wie Tokens kombiniert werden dürfen.
  • Semantik: Die Bedeutung — was der Code tatsächlich tut.

1.4 Dein erstes Python-Programm

print("Hello, World!")

print() ist eine eingebaute Funktion, die Text ausgibt. Keine Import-Anweisung nötig.

Mehrere Argumente & Parameter

print("Hallo", "Welt")            # "Hallo Welt"  (Standard-Trenner: Leerzeichen)
print("A", "B", sep="-")           # "A-B"
print("Zeile 1", end=" ")          # kein Zeilenumbruch
print("Zeile 2")                    # → "Zeile 1 Zeile 2"
print()                             # nur Zeilenumbruch
  • sep: Trenner zwischen Argumenten (Standard: " ").
  • end: was nach dem letzten Wert kommt (Standard: "\n").

1.5 Indentation (Einrückung) — KEINE Klammern!

Wichtigste Python-Regel: Code-Blöcke werden durch Einrückung definiert — nicht durch { }. PEP 8 empfiehlt 4 Leerzeichen. Tabs und Spaces nie mischen! 
if True:
    print("eingerückt")     # Teil des if-Blocks
    print("auch hier")      # immer noch drin
print("außerhalb")          # nicht eingerückt → außerhalb

Falsche Einrückung → IndentationError oder TabError.

1.6 Kommentare

# Einzeiliger Kommentar
x = 5  # auch am Zeilenende

"""
Sogenannter "Docstring" — ein String,
der als Kommentar verwendet wird (nicht offiziell,
aber Konvention für Doku am Funktions-Anfang).
"""

Python kennt nur einzeilige Kommentare mit #. Mehrzeilige „Kommentare” sind eigentlich nur ungenutzte Strings.

1.7 Literale

Ein Literal ist ein Wert direkt im Quellcode.

42              # int-Literal
3.14            # float-Literal
2.5e3           # 2500.0 — wissenschaftliche Notation
True            # bool-Literal (Großbuchstabe!)
False
None            # "nichts" — wie null in JS
"Hallo"         # str-Literal
'Welt'          # auch str
[1, 2, 3]       # list-Literal
(1, 2, 3)       # tuple-Literal
{1, 2, 3}       # set-Literal
{"a": 1}        # dict-Literal

1.8 Numeralsysteme (Zahlenbasis)

decimal = 255
binary  = 0b11111111      # = 255
octal   = 0o377           # = 255
hex     = 0xff            # = 255

# Konvertierung zurück:
bin(255)                  # '0b11111111'
oct(255)                  # '0o377'
hex(255)                  # '0xff'
Präfixe: 0b = binär, 0o = oktal, 0x = hexadezimal. Keine Präfixe bei Dezimal.

1.9 Python-Keywords (reserviert)

Diese 35 Wörter darfst du nicht als Bezeichner verwenden:

False, None, True, and, as, assert, async, await, break, class, continue, def, del, elif, else, except, finally, for, from, global, if, import, in, is, lambda, nonlocal, not, or, pass, raise, return, try, while, with, yield

Python ist case-sensitive: True ≠ true.

1.10 Bezeichner-Regeln

  • Erlaubt: Buchstaben (a–Z), Ziffern (0–9), Unterstrich _.
  • Dürfen nicht mit einer Ziffer beginnen.
  • Case-sensitive.
  • Keine Keywords.
  • Konvention: snake_case für Variablen/Funktionen, PascalCase für Klassen, UPPER_CASE für Konstanten.
my_var = 10       # ✓ snake_case (Konvention)
_private = 5      # ✓ (Unterstrich am Anfang → "privat")
__dunder__ = 1    # ✓ (doppelte Unterstriche: reserviert für Python)
2fast = 10        # ✗ SyntaxError
my-var = 10       # ✗ (Bindestrich = Minus!)
class = 1         # ✗ reserviertes Wort

1.11 Python-Versionen

Aktuell ist Python 3. Python 2 ist seit 2020 end-of-life.

Prüfen: python --version oder im Code:

import sys
print(sys.version)
