Einführung & Programmiergrundlagen

Was ist JavaScript? Wo läuft es? Syntax, Kommentare, Literale, erste Ausgaben.

1.1 Was ist JavaScript?

JavaScript (JS) ist eine high-level, interpretierte, dynamisch typisierte Programmiersprache. Sie wurde 1995 von Brendan Eich bei Netscape entwickelt und ist heute die Sprache des Webs.

  • High-level: Du musst dich nicht um Speicherverwaltung kümmern.
  • Interpretiert: Der Code wird zur Laufzeit ausgeführt, kein separates Kompilieren.
  • Dynamisch typisiert: Variablen können ihren Typ wechseln.
  • Multi-Paradigma: imperativ, objektorientiert, funktional.
Merke: JavaScript hat nichts mit Java zu tun. Der Name war Marketing.

1.2 Wo läuft JavaScript?

JS läuft überall dort, wo eine JavaScript-Engine vorhanden ist:

  • Browser (V8 in Chrome/Edge, SpiderMonkey in Firefox, JavaScriptCore in Safari)
  • Node.js — JS auf dem Server (nutzt V8)
  • Deno / Bun — moderne Alternativen zu Node
  • Eingebettet in Anwendungen, IoT-Geräten, Datenbanken (z. B. MongoDB)

Für die JSE-Prüfung wichtig: JS läuft typischerweise im Browser oder in Node.js.

1.3 Erstes Programm

Das klassische „Hello World":

console.log("Hello, World!");

console.log() gibt Text in der Konsole aus (im Browser: F12 → Console, in Node: Terminal).

Ausführen im Browser

JS kann man in HTML einbetten:

<script>
  console.log("Hallo aus der Konsole");
</script>

Oder als externe Datei:

<script src="main.js"></script>

1.4 Anweisungen (Statements)

Ein JS-Programm besteht aus Anweisungen. Jede Anweisung kann mit einem Semikolon ; enden.

let name = "Ali";
console.log(name);
let alter = 25;
ASI (Automatic Semicolon Insertion): JS fügt in vielen Fällen automatisch Semikolons ein. Trotzdem solltest du sie setzen, weil es Mehrdeutigkeiten geben kann.

1.5 Kommentare

Kommentare werden von der Engine ignoriert. Zwei Arten:

// Einzeiliger Kommentar

/*
  Mehrzeiliger
  Kommentar
*/

let x = 10; // auch am Zeilenende möglich

Nutze Kommentare, um das Warum zu erklären — nicht das Was (das zeigt der Code).

1.6 Literale

Ein Literal ist ein fester Wert, der direkt im Quellcode steht:

42              // Number-Literal
3.14            // Number-Literal (Gleitkomma)
"Hallo"         // String-Literal
'Welt'          // String-Literal (einfache Anführungszeichen)
`Template`      // Template-Literal (Backticks)
true            // Boolean-Literal
false           // Boolean-Literal
null            // null-Literal
[1, 2, 3]       // Array-Literal
{ a: 1 }        // Object-Literal

1.7 Groß-/Kleinschreibung & Bezeichner

  • JavaScript unterscheidet Groß-/Kleinschreibung: name ≠ Name.
  • Bezeichner dürfen enthalten: Buchstaben, Ziffern, _, $.
  • Dürfen nicht mit einer Ziffer beginnen.
  • Keine reservierten Schlüsselwörter (let, return, class…) als Namen.
  • Konvention: camelCase für Variablen/Funktionen, PascalCase für Klassen.

1.8 Ausgaben: console.log, alert, document.write

console.log("In der Konsole");    // Entwicklerkonsole
alert("Popup-Fenster");           // Nur Browser — modaler Dialog
document.write("In die Seite");   // Nur Browser — fügt Text in das HTML ein

Fürs Lernen und Debuggen ist console.log die beste Wahl.
